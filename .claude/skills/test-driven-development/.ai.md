# test-driven-development/

## 목적
TDD (Red-Green-Refactor) 방법론 강제 스킬. 모든 기능 구현과 버그 수정에 테스트 우선 작성을 요구한다.

## 파일
- `SKILL.md` — TDD 핵심 규칙, Red-Green-Refactor 사이클, 검증 체크리스트
- `testing-anti-patterns.md` — 테스트 안티패턴 5종 (목 행위 테스트, 프로덕션 전용 메서드, 의존성 미이해 목킹, 불완전 목, 사후 테스트)

## 핵심 원칙
"NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST" — 실패하는 테스트 없이 프로덕션 코드를 작성하지 않는다.
