# .claude/commands/

## 목적
Claude Code 슬래시 커맨드 파일 모음. 이슈 워크플로우 자동화 커맨드와 단축 별칭을 포함한다.

## 파일 목록

### 워크플로우 커맨드
- `backlog-issue.md`   — 새 이슈를 Backlog에 생성 (feat/chore 템플릿 + 구현 플랜 섹션)
- `start-issue.md`    — 이슈 작업 시작 (워크트리·브랜치·work 폴더 생성, 이슈 assign)
- `plan.md`           — 현재 이슈의 구현 계획 작성 (01_plan.md 저장)
- `remind-issue.md`   — 세션 재시작 시 AC 달성 현황 출력 + 작업 내역 기록
- `finish-issue.md`   — 구현 완료 후 커밋 및 PR 생성 (코드 간소화→리뷰→AC 체크→커밋)
- `drop-issue.md`     — 이슈 중도 포기 (사유 입력, 이슈 닫기, done 이동, 워크트리 정리)
- `cleanup-issue.md`  — PR 머지 후 워크트리·로컬+리모트 브랜치 정리
- `update-changelog.md` — scope별 CHANGELOG.md 주차별 업데이트

### 단축 별칭
- `bi.md` — `/backlog-issue` 단축 별칭
- `si.md` — `/start-issue` 단축 별칭
- `fi.md` — `/finish-issue` 단축 별칭
- `ci.md` — `/cleanup-issue` 단축 별칭
- `ri.md` — `/remind-issue` 단축 별칭

## 규칙
- 커맨드 파일은 YAML frontmatter의 `description` 필드를 반드시 포함한다
- 단축 별칭은 독립 파일로 구현한다 (symlink 사용 금지)
- 별칭 파일의 `description`은 원본과 구분되는 "단축 별칭" 형식으로 작성한다
