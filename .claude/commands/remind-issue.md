## 인수 형식 (모두 선택사항)

`$ARGUMENTS`는 다음 형식 중 하나거나 생략 가능하다:

| 형식 | 예시 | 동작 |
|------|------|------|
| 생략 | (없음) | 현재 브랜치에서 자동 감지 |
| 이슈번호만 | `15` | 이슈번호로 브랜치 탐색 |
| 6자리 패딩+짧은이름 | `000015-my-feature` | 직접 경로 구성 |
| 짧은이름만 | `my-feature` | 브랜치 목록에서 매칭 |

## 실행 순서

### 1. 브랜치·이슈번호 확정

```
git branch --show-current
```
결과 예: `feat/000015-my-feature` → PADDED=`000015`, 짧은이름=`my-feature`, 이슈번호=`15`

### 1.5. 터미널/tmux 윈도우 컨텍스트 설정

```bash
if [ -n "$TMUX" ]; then
  tmux rename-window -t "$TMUX_PANE" "{이슈번호}-{짧은이름}"
else
  printf '\033]0;#%s %s\007' "{이슈번호}" "{짧은이름}"
fi
```

### 2. AC 목록 추출

`{WORKFOLDER}/00_issue.md`의 `## 완료 기준` 섹션 또는 이슈 body에서 추출.

### 2.5. 구현 계획 확인 및 자동 작성

`01_plan.md`의 `## 구현 계획`이 비어 있으면 `/plan` 커맨드를 실행해 채운다.

### 3. AC 자동 체크

미완료(`[ ]`) AC 항목 각각에 대해 완료 여부를 판단한다:
1. 파일 존재 여부 확인
2. `git log` 커밋 메시지에서 증거 확인
3. `git status` 변경사항 분석
4. 실제 파일 내용 확인

증거가 명확할 때만 `[x]`로 변경. 불확실하면 `[ ]` 유지.

### 4. 현재 단계 판단

1. 미커밋 변경사항 있음 → **"구현 중"**
2. 미커밋 없고 AC 미완료 → **"구현 대기"**
3. AC 전체 완료 → **"완료 — /finish-issue 실행 필요"**

### 5. 현황 출력

```
────────────────────────────────────────
이슈 #{번호} — {제목}
브랜치: {브랜치명}
────────────────────────────────────────
AC 진행 상황 ({완료수}/{전체수} 완료):
  [x] 완료된 항목
  [ ] 미완료 항목

📍 현재 단계: {구현 중 | 구현 대기 | 완료}

🔀 최근 커밋 (git log --oneline -3):
  {커밋 목록}

📁 미커밋 변경사항:
  {변경 목록, 없으면 "없음"}
────────────────────────────────────────
```

### 6. 작업 내역 기록

`00_issue.md`의 `## 작업 내역` 섹션에 현황 스냅샷 추가 (오늘 날짜 항목이 있으면 업데이트).
