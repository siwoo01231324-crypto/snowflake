# pipelines/ — Snowpark 전처리 파이프라인

## 목적
Snowflake Marketplace 뷰를 기반으로 분석 마트를 구성하는 Snowpark Python 파이프라인.
다양한 데이터 소스(아정당, SPH, RICHGO)를 통합하여 25개 구 × 월 단위 통합 마트를 생성한다.

| `__init__.py` | #21 | 패키지 초기화 |
| `preprocessing.py` | #21 | `build_integrated_mart()` 함수 (핵심) |

## 주요 함수

### `build_integrated_mart(session: Session) -> DataFrame`

**목적**: 25개 구 × 월 spine 기반 통합 마트 생성

**입력**: Snowpark `Session` 객체

**출력**: `MOVING_INTEL.ANALYTICS.MART_MOVE_ANALYSIS` 테이블 (Snowflake 저장 + DataFrame 반환)

**핵심 로직**:

1. **Spine 생성**: 25개 구(M_SCCO_MST) × 아정당 월 범위 (CROSS JOIN)
2. **데이터 수집** (각각 구 단위 집계):
   - 아정당 V_TELECOM_DISTRICT_MAPPED (25개 구, 월별)
   - SPH 유동인구 (3개 구: 중구, 영등포구, 서초구)
   - SPH 카드매출 (3개 구)
   - SPH 자산소득 (3개 구)
   - RICHGO 시세 (3개 구)
3. **조인**: spine에 LEFT JOIN으로 모든 팩트 연결
4. **DATA_TIER 파생**: 
   - `MULTI_SOURCE` (3개 구 = 4종 데이터 모두 유효)
   - `TELECOM_ONLY` (22개 구 = 아정당만 유효)
5. **저장**: `MART_MOVE_ANALYSIS` 테이블 (overwrite)

## 데이터 출처별 범위

### MULTI_SOURCE (3개 구)
- **구**: 중구(11140), 영등포구(11560), 서초구(11650)
- **데이터**: 아정당 + SPH + RICHGO (4종 신호 모두 유효)
- **용도**: 완전한 분석 가능, #22 MOVE_SIGNAL_INDEX 고품질 계산

### TELECOM_ONLY (22개 구)
- **구**: 위의 3개 구를 제외한 나머지 22개 구
- **데이터**: 아정당만 유효 (OPEN_COUNT, CONTRACT_COUNT, PAYEND_COUNT)
- **용도**: 이사 신호(S1)만 가능, 다른 신호는 NULL
- **#22 대응**: DATA_TIER 기반 차등 가중치 적용

## 불변식

1. **테이블명**: `MART_MOVE_ANALYSIS` (NOT INTEGRATED_MART)
2. **해상도**: 구(CITY_CODE) 단위 × 월(STANDARD_YEAR_MONTH) 단위 (동 단위 아님)
3. **Spine**: 25개 구 × 아정당 월 범위 (CROSS JOIN으로 생성)
4. **DATA_TIER**: 명시적으로 'MULTI_SOURCE' / 'TELECOM_ONLY' 구분
5. **년월 형식**: STANDARD_YEAR_MONTH는 `YYYYMM` 문자열 (DATE 아님)
6. **왼쪽 조인**: 모든 팩트는 LEFT JOIN (NULL 허용)
7. **#22 호환성**: 12개 필수 컬럼 전부 포함
