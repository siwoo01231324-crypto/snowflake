## 1. (성지욱) Snowflake 해커톤 2026 Top 3 — Next Gen Index

**영상**: https://snowflake.wistia.com/medias/4i0gvjsg5g  
**행사**: Snowflake User Group Meet up (2026-03-17)  
**발표자**: 성지욱 (SK AX, 디지털 트윈 팀, 반도체 제조 도메인)

### 요약
SK AX 디지털 트윈 팀의 성지욱님이 Snowflake 해커톤 2025에서 개발한 **"Next Gen Index"** 서비스를 발표. 20~40세 여성 대비 유아 인구 비율이라는 특이 데이터에 주목해, **DEA(Data Envelopment Analysis, 데이터 포락 분석)** 기법으로 지역별 도시 효율성을 측정·비교하는 서비스를 구현했다. 단순 분석을 넘어 인풋 파라미터를 조절하는 시뮬레이션 기능을 추가해 정책적 방향성 도출을 목표로 했다.

### 주요 포인트
- **아이디어 출발점**: 일반적이지 않은 유아 인구 비율 데이터에서 영감 → "가족 친화 도시 효율성" 분석 서비스로 발전
- **분석 모델 — DEA**: 인풋(대출액, 고소득자 수, 백화점 매장 수, 교육 시설 수) 대비 아웃풋(총 소비 지출, 백화점 매출, 여성당 유아 인구 비율)의 효율성 측정
- **결과**: 영등포구 효율 최하(0.13점), 서초구·중구 만점(1.5점) → 표본 3개 구로 상대평가여서 한계 존재
- **시뮬레이션 기능**: 인풋 파라미터 조절 시 효율성 변화 즉시 확인 → 정책 방향성 도출 가능
- **NetworkX 활용**: 변수 간 상관관계를 네트워크 그래프로 시각화 (임계값 조절 가능)
- **기술 스택**: Streamlit 대시보드, NetworkX, Snowflake 데이터 연동
- **자기 성찰**: Snowflake 고급 기능(생성 AI, 자동 보고서 등) 미활용이 아쉬움 → 다음 해커톤 참가자들에게 제공 기능 100% 활용 권고

### 키워드 & 개념
`DEA(데이터 포락 분석)` `Next Gen Index` `도시 효율성` `유아 인구 비율` `시뮬레이션` `NetworkX` `Streamlit` `Snowflake 해커톤 2025`

### 인사이트
> 데이터 분석의 완성도보다 **Snowflake 플랫폼 기능 활용도**가 해커톤 평가에서 중요한 비중을 차지함. 분석 정확도와 UX 편의성을 함께 고려해야 하며, 제공된 모든 기능을 활용하는 전략이 유리하다.

## 2. (박준호) 아이디어에서 구현까지 단숨에 — 백화점 데이터로 주식 투자 지표 만들기

**영상**: https://snowflake.wistia.com/medias/80p2bxdtee  
**행사**: Snowflake User Group Meet up (2026-03-17)  
**발표자**: 박준호 (CDO)

### 요약
CDO로 일하는 박준호님이 2025 Snowflake 해커톤(Streamlit 데이터 앱 부문)에서 개발한 **백화점 방문자 수·카드 소비 데이터 기반 주식 투자 지표 서비스**를 발표. "남들이 안 보는 지표"를 찾다가 위치기반 데이터(로플렛)의 백화점 방문자 수와 SPH 카드 소비내역에 주목해, 이를 주가 데이터와 연계한 백테스팅 서비스를 구현했다. 코스피 30% 하락장에서도 10~20% 수익률을 기록하는 전략을 검증했다.

### 주요 포인트
- **아이디어 출발점**: 주식 투자 시 "남들이 안 보는 지표" 탐색 → 백화점 방문자수 + 카드 소비내역으로 주가 선행 지표 가설 수립
- **데이터 소스**: 로플렛(백화점 방문자수), SPH(카드 소비), 공공데이터(주가 일봉 OHLCV)
- **Snowflake Marketplace 활용**: 클릭 한 번으로 데이터 적재 — ETL 부담 최소화
- **개발 흐름**: 입수 → 정제(결측치·이상치 처리) → 집계·조인 → Streamlit 시각화 → AI 분석
- **백테스팅 기능**: 방문자 증가율 임계값 설정 → 매수 시그널 생성 → 손익 시뮬레이션 (손절 -2%, 익절 +4%)
- **박스플롯 수익률 비교**: 단기(1~3개월) vs 장기(6~12개월) 투자 성과 시각화
- **LLM 투자 인사이트 챗봇**: Claude 3.5 연동 → 백테스팅 결과 + 시장 상황 기반 투자 전략 자동 생성
- **비즈니스 모델 구상**: 데이터 업체 협약 → Snowflake Marketplace 상품화 + GCP/AWS 퀴닝 시스템 연동 API화

### 키워드 & 개념
`백테스팅` `위치기반 데이터` `Snowflake Marketplace` `Streamlit` `Claude API` `주식 투자 지표` `박스플롯` `손절/익절 전략`

### 인사이트
> **일반적이지 않은 데이터(방문자 수)** 를 주식 지표로 활용하는 발상 전환이 핵심. Snowflake Marketplace의 원클릭 데이터 연동은 해커톤에서 개발 시간을 크게 절약하는 전략적 선택이었으며, AI 분석 기능 추가로 UX 완성도를 높인 점이 인상적.

## 5. (전수범) Real World AI — 데이터 사일로 격파기

**영상**: https://snowflake.wistia.com/medias/l502mozf2u  
**행사**: 유저그룹 웨비나 (2026-04-01)  
**발표자**: 전수범 (풀무원 AI 데이터 아키텍트, 2026 Snowflake SuperHero)

### 요약
풀무원 AI 데이터 아키텍트 전수범님이 **실제 SCM 환경의 데이터 사일로 문제**를 Snowflake로 해결한 사례를 발표. "3월 매출이 얼마인가?"라는 단순 질문에 각 부서가 서로 다른 숫자를 제시하는 현실에서 출발해, Snowflake 기반 단일 데이터 플랫폼을 구축하고 AI 에이전트를 통한 의사결정 자동화까지 구현한 여정을 공유했다.

### 문제 정의 — 데이터 사일로
- **Single Source of Truth 부재**: "3월 매출"에 재무(출고 기준 12억), 영업(판매 시점 10억), 경영기획(입금 기준) 등 부서마다 다른 숫자
- **데이터 분산**: WMS·OMS·ERP·CRM이 법인별로 따로 운영 → 8개 법인 × n개 시스템
- **비표준화**: 시스템마다 컬럼 정의·메트릭 기준이 달라 집계 오류 다발
- **레거시 한계**: 대용량 처리 불가, 데이터 추출에 과도한 시간 소요
- **거버넌스 부재**: 데이터 접근 권한 요청만 수 주 소요

### 아키텍처 — Snowflake 올인원 플랫폼
- **Medallion Architecture**: Bronze(원본 적재) → Silver(정제·표준화·통합) → Gold(의사결정용 KPI·대시보드)
- **Semantic Layer 추가**: Gold 위에 SCM 핵심 도메인(제고·수요·분급) 기반으로 AI 에이전트가 바로 소비할 수 있는 형태로 구조화
- **Private Share 활용**: 다른 팀의 Snowflake 계정에서 데이터 공유 수신

### DX 전략 3단계
1. **대시보드**: 성과지표 + 과정지표 동시 관리
2. **비즈니스 애널리틱스**: 시나리오 분석, 루트코즈 분석 (무엇이 → 왜 발생했는지)
3. **SCM Intelligence**: AI가 계획·실행·학습 반복 → 수요 예측 기반 생산·재고 자동 최적화

### 실제 유스케이스
**① 미국 법인 PO 컨테이너 최적화**
- 기존: 엑셀 수기 작업 → 긴급 변경 시 전체 재작업 → **2주 소요**
- 개선: Cortex Analyst로 비즈니스 로직 YAML 정의 → Cortex Code로 Python 엔진 자동 생성 → AI 에이전트 채팅으로 즉시 수정 실행
- 결과: **1초 클릭으로 처리, 업무 리소스 99.9% 절감**

**② SKU Deletion 의사결정 자동화**
- 기존: 매출 기준 단순 판단 (매출↑=유지, 매출↓=삭제) → 수익성 누락
- 개선: 재무(매출원가·영업익) + SCM(물류비·보관비·재고회전율) 통합 → AI 에이전트가 복합지표 자동 계산 → 유지/조정/삭제 시나리오 자동 제안

### 키워드 & 개념
`데이터 사일로` `Single Source of Truth` `Medallion Architecture` `Semantic Layer` `Cortex Analyst` `Cortex Code` `AI 에이전트` `SCM Intelligence` `풀무원` `컨테이너 최적화` `SKU Deletion`

### 인사이트
> **"AI 에이전트 이전에 Semantic Layer가 먼저다."** 좋은 데이터 구조(Garbage In → Garbage Out 방지)가 AI 활용의 선결 조건. 해커톤에서도 데이터를 시맨틱 레이어로 잘 정의하는 것이 프로젝트 성패를 가른다는 실전 교훈.

Snowflake Data SuperHero 성순모님이 **2026 AI 트렌드**와 **Snowflake Cortex AI 10가지 핵심 스킬**을 발표. 특히 Cortex Code("코코")를 중심으로 해커톤에서 활용 가능한 스킬을 실전 데모와 함께 소개했다. "가장 강력한 모델보다 코코가 해커톤 우승을 결정한다"는 메시지로 마무리.

### 2026 AI 주요 트렌드
1. **에이전트 중심 전환**: 기업 85%가 AI 커스터마이징 계획 → 비즈니스 맞춤형 에이전트 시대
2. **POC → 프로덕션 가속화**: 현재 25% → 6개월 내 50%+ 프로덕션 전환 예상
3. **프로세스·업무 재설계**: 60%+ 기업이 AI 기반 프로세스 트랜스포메이션 진행 중
4. **통합 에코시스템화**: 단일 모델 → 에이전트 협업 → 하나의 통합 시스템으로 진화
5. **기술보다 유스케이스가 과제**: 기술 완성도보다 실제 활용 사례 발굴이 핵심 숙제

### Snowflake Cortex = 데이터 중심 통합 AI 플랫폼
모델 + 거버넌스 + 에이전트 + Analyst + Search + ML을 하나의 플랫폼에서 제공

### Cortex Code("코코") 10가지 스킬
| 스킬 | 설명 | 해커톤 활용도 |
|------|------|-------------|
| **AI Functions** | SQL 구문으로 감정분석·PII 마스킹 등 즉시 실행 | ★★★ |
| **Cost Intelligence** | 웨어하우스·쿼리별 비용 분석·이상탐지 자동화 | ★★ |
| **Data Clean Room** | 데이터 협업·고객 오버랩 분석 | ★ |
| **Data Governance** | PII 감지·마스킹·리역검사·컴플라이언스 | ★★ |
| **Data Quality** | DMF 기반 품질 메트릭 자동 모니터링 | ★★ |
| **dbt Project** | dbt 프로젝트 실행·조회·수정 자연어 제어 | ★★ |
| **Integrations** | AWS S3·Azure Blob 통합 파이프라인 코드 생성 | ★★★ |
| **Lineage** | 데이터 출처·다운스트림 영향도 추적 | ★★ |
| **Machine Learning** | 모델 학습·레지스트리·Feature Store·모니터링 | ★★★ |
| **Notebook Workspace** | 노트북 생성·셀 편집·탐색적 분석 자동화 | ★★★ |

### 실전 데모 흐름 (자연어만으로)
1. 스키마 파악 → 테이블 리스트 조회
2. 비즈니스 인사이트 질문 → 쿼리 5개 자동 생성·검증
3. 쿼리 실행 결과 → 노트북 셀 자동 생성·시각화
4. 에러 발생 시 자동 감지·코드 수정
5. **Streamlit 앱으로 자동 배포** (코드 작성 없이)

### 키워드 & 개념
`Cortex Code(코코)` `AI Functions` `Notebook Workspace` `Integrations` `Machine Learning` `에이전트 중심 전환` `POC→프로덕션` `Streamlit 자동 배포`

### 인사이트
> **"해커톤 우승은 가장 강력한 모델이 아니라 코코(Cortex Code)를 얼마나 잘 활용하느냐가 결정한다."** 자연어로 쿼리 생성 → 노트북 시각화 → Streamlit 앱 배포까지 원스톱으로 가능한 코코 활용이 개발 공수를 극적으로 줄여준다.
