## 섹션 1. Executive Summary

### 프로젝트 한 줄 정의

**무빙 인텔리전스(Moving Intelligence)**는 통신 주소변경 데이터를 이사(移徙)의 선행지표로 활용하여, 이사가 일어나기 **2~4주 전에** 수요를 예측하고 B2B 고객사에 제공하는 이사 사전 예측 플랫폼이다.

### 핵심 문제

한국에서는 연간 약 628만 명이 이사를 한다 (출처: 통계청 2024 국내인구이동통계). 이사는 1건당 평균 200~500만원의 소비를 수반하는 인생 최대의 소비 이벤트이지만, 이사업체·가전렌탈사·인테리어 플랫폼·통신사 등 B2B 사업자들은 **이사가 이미 일어난 후에야** 이를 인지한다. 전입신고는 이사 후 14일 이내 의무 신고이며, 기존 DMP·광고 플랫폼의 이사 세그먼트는 90일 윈도우의 AI 추정값에 불과해 정밀도가 낮다.

### 솔루션 요약

아정당의 통신 주소변경 데이터(이사 추정)와 RICHGO 인구이동 데이터, SPH 소득·소비 데이터를 Snowflake 플랫폼에서 결합·분석하여, 이사 예정 가구의 **지역·시기·세그먼트(소득 수준, 가구 규모)**를 이사 2~4주 전에 예측한다. B2B 고객사의 CRM에 API로 직접 주입하거나 Streamlit 대시보드로 제공한다.

### 핵심 가치

- **선행성**: 경쟁사 대비 유일하게 이사 전 예측 (기존 서비스는 모두 사후 데이터)
- **정밀도**: 통신 주소변경이라는 물리적 신호 기반 — AI 추정이 아닌 실제 행동 데이터
- **B2B ROI**: 생활 이벤트 트리거 캠페인은 일반 캠페인 대비 전환율 200% 높음 (출처: 글로벌 Pre-Mover 마케팅 연구)

## 섹션 2. 고객 / 문제 정의 / 가치 제안 (VPC)

### 2-1. 타겟 고객 세그먼트 및 JTBD

무빙 인텔리전스의 핵심 고객은 **B2B 사업자**다. MVP는 B2B 수익에 집중하며, B2C(개인 이사 예정자) 서비스는 Phase 3 이후 확장 단계에서 진출한다.

**B2B 핵심 JTBD**: "이사할 사람을 미리 알고, 경쟁사보다 2~4주 먼저 선제적으로 서비스를 제안하고 싶다"
**B2B 핵심 Pain**: "이사 수요를 사후적으로만 파악해 고객 확보 기회를 반복적으로 상실한다 — CAC 상승 악순환"

#### Primary 고객: 이사 연관 서비스 B2B 사업자

| 세그먼트 | 대표 기업 | 핵심 JTBD | 현재 Pain |
|---------|---------|---------|---------|
| 가전 렌탈사 | 코웨이, 쿠쿠, LG전자, SK매직 | 이사 예정 가구에 경쟁사보다 먼저 렌탈 계약 제안 | 판관비 55~60% 수준이나 타겟팅 정밀도 낮아 CAC 높음 |
| 인테리어 플랫폼 | 오늘의집, 집닥 | 이사 예정 지역 집중 광고로 고객 획득 비용 절감 | 이사 완료 후 경쟁사에 이미 선점된 상태에서 광고 |
| 이사 O2O 플랫폼 | 짐싸, 다이사 | 다음 달 이사 수요 예측으로 차량·인력 배치 최적화 | 수요 예측 없이 관행적 배치 → 성수기 과부하·비수기 유휴 |
| 통신사 | KT, SKT, LGU+ | 이사 시 인터넷·TV 해지 고객을 사전 파악하여 이탈 방지 | 이사 완료 후 해지 신청이 들어와야 인지 |
| 부동산 중개 플랫폼 | 직방, 다방 | 이사 수요 증가 지역 매물 집중 노출 | 이사 수요와 무관한 광고 집행, 전환율 낮음 |

| 부동산 투자자 | 특정 지역 이사 유입·유출 트렌드로 투자 타이밍 결정 |

### 2-4. 핵심 차별화

**"이사 2~4주 전 예측"이 가능한 이유**

통신 계약 주소변경은 실제 이사보다 2~4주 앞서 발생한다. 이사 준비 단계에서 새 거주지의 인터넷·TV를 사전 개통하기 때문이다. 이는 공식 전입신고(이사 후 14일 이내)보다 최대 6~8주 빠른 신호다.

## 섹션 3. 솔루션 & 기술 아키텍처

### 3.1 솔루션 개요

**무빙 인텔리전스**는 통신 주소변경 데이터를 이사의 선행지표로 활용하여, **"누가 곧 이사하는가"를 이사 2~4주 전에 예측**하고 B2B 고객사에 제공하는 이사 수요 예측 플랫폼이다. 부동산·통신·지역경제·금융 데이터를 Snowflake 플랫폼에서 통합 분석하여, B2B 고객이 선제적 마케팅·영업에 즉시 활용할 수 있는 예측 API와 대시보드를 제공한다.

### 3.2 핵심 기능 4가지

#### 기능 1. 이사 수요 예측 (Move Demand Prediction) — MVP 핵심
- 아정당 통신 계약 주소 변경 데이터(이사 선행지표) + RICHGO 인구 전입전출 데이터를 결합
- 지역별·시기별 향후 2~4주 내 이사 예정 가구 수를 Snowpark ML 모델로 예측
- B2B 고객이 선제적 마케팅·영업에 활용할 수 있는 예측 API 제공 — "어느 지역에서 몇 가구가 이사를 앞두고 있는가"를 가구 세그먼트(소득·가구 규모·아파트 가격대)와 함께 제공

#### 기능 2. 지역 생활 점수 (Neighborhood Life Score) — Phase 2 확장
- SPH 유동인구(SKT) × 소비 트렌드(신한카드) × 소득 수준(KCB) × RICHGO 아파트 AI 시세를 결합
- 서초구·영등포구·중구 3개 구를 행정동 단위로 세분화하여 종합 생활 매력도 점수 산출
- Streamlit 지도 대시보드로 직관적 시각화

#### 기능 3. 이사 후 투자 인사이트 (Post-Move Investment Insights) — Phase 2 확장
- NextTrade 주식 데이터(지역 관련 리츠·건설주·소비재) × RICHGO 부동산 AI 시세의 시계열 상관관계 분석
- "이 지역 유동인구가 증가할 때 어느 섹터 주가가 동행/선행하는가?" 패턴 도출

#### 기능 4. AI 추천 에이전트 (Moving Intelligence Agent) — Phase 2 확장
- Cortex Agents 기반 멀티툴 에이전트: Cortex Analyst + Cortex Search + Custom UDF 오케스트레이션
- 자연어로 질문하면 4종 데이터셋을 통합 조회하여 종합 답변 생성

### 3.3 Snowflake 기술 스택 설계

#### Tier 1 — 필수 (데이터 파이프라인 기반)

| 기술 | 역할 |
|------|------|
| **Snowflake Marketplace** | 4종 데이터셋 연동. 데이터 복사 없이 계정 내 직접 쿼리 |
| **Snowpark Python** | 데이터 전처리 파이프라인, Feature Engineering, ML 모델 학습·배포 |
| **Streamlit in Snowflake** | 인터랙티브 대시보드 UI. 별도 서버 없이 완전 호스팅 |

#### Tier 2 — 권장 (AI 분석 강화)

| 기술 | 역할 |
|------|------|
| **Cortex AI Functions** | `AI_COMPLETE`: 이사 지역 맞춤 인사이트 텍스트 생성 / `AI_SENTIMENT`: 지역 커뮤니티 리뷰 감성 분석 / `AI_AGG`: 월별 소비 트렌드 요약 |
| **Cortex Analyst** | 4종 데이터셋 시맨틱 모델 정의 → 자연어 질의를 SQL로 자동 변환 |
| **VECTOR 타입 + AI_EMBED** | 아파트 단지 특성 벡터화 → 유사 단지 추천. Arctic Embed 2.0(한국어, 8,192 토큰) 활용 |

#### Tier 3 — 차별화 (최고 가산점)

| 기술 | 역할 |
|------|------|
| **Cortex Agents** | Cortex Analyst + Cortex Search + Custom UDF를 오케스트레이션. Claude Sonnet 4.6 기반 |
| **Cortex Search** | 아파트 단지 정보·지역 리포트 RAG 파이프라인. 하이브리드 검색(키워드+시맨틱) |

### 4.2 데이터 결합(Join) 전략

#### 핵심 조인 키: 행정동 코드

| 결합 대상 | 공통 키 | 결합 유형 | 난이도 |
|-----------|---------|-----------|--------|
| RICHGO ↔ SPH | 행정동/법정동 (서울 지역) | 공간 조인 | 낮음 |
| RICHGO ↔ 아정당 | 지역 (이사 출발지/도착지) | 지역 매핑 | 중간 |
| SPH ↔ 아정당 | 지역 (서초구/영등포구/중구) | 공간 조인 | 낮음 |
| NextTrade ↔ RICHGO | 시간축 (월별 집계) | 시계열 조인 | 낮음 |

**행정동 코드 기반 통합 데이터 마트 구성 예시 (Snowpark Python 구현)**

```sql
-- RICHGO (아파트) × SPH (생활지표) × 아정당 (이사 신호) 결합

SELECT
    r.apt_complex_name,
    r.region_dong_code,          -- 행정동 코드 (조인 키)
    r.ai_price_202601,
    r.ai_price_trend_6m,
    s.avg_floating_pop_2025,     -- SKT 유동인구 평균
    s.avg_consumption_2025,      -- 신한카드 소비 평균
    s.avg_income_level,          -- KCB 소득 수준
    a.move_signal_score          -- 아정당 이사 신호 지수
FROM richgo_apt_master r
JOIN sph_lifestyle_index s
    ON r.region_dong_code = s.dong_code  -- 행정동 코드 조인
LEFT JOIN ajeong_move_signal a
    ON r.region_dong_code = a.region_code
WHERE s.district IN ('서초구', '영등포구', '중구')  -- SPH 커버리지 제한 준수
```

#### 시계열 시차 조인 (이사 선행지표 검증)

```
아정당 이사추정 (T)
    ─ lag +2~4주 ─→ RICHGO 인구이동 공식 신고 (T+n)
    ─ lag +4~8주 ─→ SPH 유동인구 변화 (T+m)
    ─ lag +동행  ─→ NextTrade 관련주 거래량 (T~T+2)
→ 시차 상관 분석으로 예측력 가장 높은 lag 도출
```

### 4.4 Data Analysis Insight 3개

#### Insight 1. 이사 수요 예측: 통신 주소변경이 B2B 고객의 골든타임을 만든다

**활용 데이터**: 아정당(이사 추정) + RICHGO(인구 이동자수)

**분석 방법**
- Snowpark Python으로 두 시계열을 월별 집계
- 시차 상관분석(lag correlation): lag = 0~8주 범위에서 피어슨 상관계수 산출
- 상관계수가 최대인 lag 값을 선행 주기로 설정, 지역별·세그먼트별 예측 수요 산출

**주요 효과**
- 이사 수요를 공식 데이터 대비 약 3~4주 조기 탐지 → B2B 고객에게 경쟁사보다 먼저 접촉할 골든타임 제공
- 이사 관련 서비스(통신·렌탈 가전·인테리어) 수요를 선제적으로 매칭하여 전환율 개선
- 지역별·시기별 이사 예정 가구 수 예측을 API로 제공 → B2B 고객의 운영 자원(인력·재고·광고 예산) 사전 최적화

#### Insight 2. 유동인구-소비-시세 삼각 연관: 사람이 모이면 소비가 늘고 집값이 오른다

**활용 데이터**: SPH(유동인구 + 소비 + 소득, 서초구·영등포구·중구) + RICHGO(아파트 AI 시세)

**분석 방법**
- 행정동 코드 기반 공간 조인으로 4종 지표 통합
- 월별 시계열로 유동인구 증감률 → 소비 증감률 → 시세 변동률 순차 패턴 분석
- `AI_AGG`로 지역별 트렌드 자동 요약, Streamlit 지도로 시각화

**주요 효과**
- 지역 투자 매력도 점수("생활 점수") 산출 근거 확보
- 유동인구 증가 선행 지역을 조기 포착하여 투자 시그널로 활용
- SPH 3개 구 내 동별 미시적 상권 격차 정량화

#### Insight 3. 이사 시즌과 주식 섹터 연동: 건설·소비재 거래량은 이사 수요에 동행한다

**활용 데이터**: NextTrade(건설·리츠·소비재 관련주) + 아정당(이사 추정 월별 지수) + RICHGO(인구 이동)

**분석 방법**
- NextTrade 체결 데이터에서 건설·부동산 섹터 종목 필터링
- Snowpark Python으로 이사 신호 지수 × 섹터 거래량 시계열 상관 분석
- 계절성 분해(decomposition)로 이사 성수기(봄·가을) 구간 특이점 추출

**주요 효과**
- "이사 수요 증가 → 건설주 거래량 선행 상승" 패턴 실증
- 이사 후 투자 포트폴리오 리밸런싱 제안의 데이터 근거 확보
- 주식·부동산 통합 자산 전략 수립 지원

### 5-1. 수익 모델

> **수익 구조 방향**: B2B 이사 수요 예측 데이터 판매가 핵심 수익원이다. MVP 단계에서는 이사 수요 예측 API / SaaS 대시보드를 통한 B2B 수익에 집중한다. B2C(개인 소비자 대상) 서비스는 Phase 3 이후 확장 단계에서 검토한다.

#### ① 이사 수요 예측 API / SaaS 대시보드 (핵심 수익원 — B2B)

| 플랜 | 월 요금 | 주요 제공 내용 | 타겟 고객 |
|------|--------|--------------|---------|
| **Starter** | 100만원 | 수도권 1개 구 이사 수요 지수 + 월간 리포트 | 영세 이사업체, 소형 인테리어 업체 |

### 7-3. 차별화 포인트 4가지

**① 4개 데이터셋 결합**

```
통신 주소변경 신호 (선행지표)
    +
부동산 실거래가 데이터 (소득 프록시)
    +
인구통계 데이터 (세대원 수·연령대)
    +
이사 서비스 수요 데이터 (확정 이벤트)
    ↓
이사 2~4주 전 예측 + 세그먼트 정밀화
```

**② Snowflake 네이티브 아키텍처**

고객사 ETL 파이프라인 불필요 — Snowflake Data Share로 즉시 CRM 연동. 데이터 엔지니어 리소스 없이도 사용 가능. (IT 부서 Job: "안정적이고 유지보수 불필요한 데이터 피드")

**③ 이사 전 예측 = 한국 전세·월세 특화**

한국은 부동산 매물 등재(MLS) 없이 전세·월세 이동이 발생 → 미국식 부동산 매물 기반 Pre-Mover 데이터 적용 불가. 통신 주소변경 데이터가 유일한 선행 시그널. 이 구조적 특성이 진입 장벽이자 독점 모트다.

**④ APAC 시장 퍼스트무버**

APAC 전역에 Pre-Mover 전문 B2B 서비스 전무 (출처: 13_competitive-landscape.md). 한국 연간 이사 건수 628만명(통계청 2024), 이사 연관 시장 50조원+ 규모의 공백 시장 선점.

### 7-4. 포지셔닝 맵

```
                    높은 예측 선행성
                         |
              무빙 인텔리전스 (목표)
              [이사 2~4주 전 / CRM API]
                         |
낮은 데이터   ─────────────────────── 높은 데이터
결합 수준    KT K-Ads   SKT지오비전    결합 수준
                         |
              IGAWorks (90일 추정, 광고만)
                         |
                    낮은 예측 선행성

X축: 데이터 결합 수준 (단일 ← → 다중 소스 결합)
Y축: 예측 선행성 (사후 집계 ← → 이사 수주 전 예측)
```

무빙 인텔리전스의 목표 포지션: **가장 높은 예측 선행성 + 가장 높은 데이터 결합 수준.** 기존 어떤 플레이어도 점유하지 않은 사분면.
