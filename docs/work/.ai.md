# docs/work/

## 목적
이슈별 작업 내역을 관리한다. 진행 중인 작업은 `active/`, 완료된 작업은 `done/`에 보관한다.

## 구조
| 디렉토리 | 역할 |
|----------|------|
| `active/` | 현재 진행 중인 작업 폴더 |
| `done/` | 완료(머지) 또는 포기된 작업 폴더 |

## 폴더 이동 규칙
- `active/` → `done/`: `/finish-issue` 또는 `/cleanup-issue` 커맨드가 자동 처리
- 중도 포기: `/drop-issue` 커맨드 → `done/DROPPED-{폴더명}` 으로 이동
