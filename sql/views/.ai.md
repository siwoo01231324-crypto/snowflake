# sql/views — Marketplace 연동 뷰

## 목적
Snowflake Marketplace 원본 데이터를 MOVING_INTEL.ANALYTICS 스키마에서 접근 가능하게 하는 참조 뷰 모음.
데이터 복사 없이 원본 DB를 직접 참조한다 (SELECT ... FROM {Marketplace DB}).

## 네이밍 컨벤션
- `{번호}_{데이터셋}_views.sql` — 번호는 실행 순서 (001=DDL, 002=RICHGO, 003=SPH, ...)
- 뷰 이름: `V_{데이터셋}_{도메인}` (예: V_RICHGO_MARKET_PRICE)

## 불변식
- 조인 키: `PROVINCE_CODE(2) + CITY_CODE(5) + DISTRICT_CODE(8)`
- `V_SPH_REGION_MASTER.DISTRICT_GEOM` — GEOGRAPHY 타입, NOT NULL
- 서울 25개 구 467개 동 전체 커버
- 아정당은 시/도 + 시/군/구 단위 (동 단위 아님)
- V05 OPEN_COUNT가 이사 프록시의 핵심 시그널
- NextTrade는 시간축 조인만 (행정동 조인 없음)

## 의존성
- 모든 뷰는 `sql/ddl/001_create_database_schema.sql` (#16)이 먼저 실행되어야 함
